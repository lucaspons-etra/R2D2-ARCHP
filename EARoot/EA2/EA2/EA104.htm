Scenario Type
Basic Path Basic Path
Details:
Steps:
1. DER sends measurements to DER SCADA<br/>
2. DER SCADA sends measurements to flexibility system<br/>
3. flexibility system calculates available flexibility from distributeds energy resources<br/>
4. RTU sends grid values to SCADA<br/>
5. SCADA sends grid values to EMS<br/>
6. EMS calculates and forecasts grid problems  active or in ther near future<br/>
7. EMS requests flexibility from flexibility system<br/>
8. flexibility system returns available flexibility as requested to EMS<br/>
9. EMS request the activation of flexibility to the flexibility system<br/>
10. flexibility system sends control commands to DER SCADA<br/>
11. DER SCADA sends commands to DER<br/>
12. RTU sends new measurements to SCADA<br/>
13. SCADA sends new measurements to EMS<br/>
14. EMS assess the grid problem is solved<br/>
Exceptions:
11a. Assuring of secure communication channel between devices and system from Scenario 1 and procedures in case of attack (at End)
Assuring of secure communication channel between devices and system from Scenario 1 and procedures in case of attack Alternate
Details:
Steps:
1. EMS sends flexibility request details to security enforcer <br/>
2. security enforcer checks the security of the requrest and the possibility of being a cyber threat<br/>
3. security enforcer sends the cancellation of the flexibility request to EMS<br/>
4. security enforcer sends the threat details to cyber security system<br/>

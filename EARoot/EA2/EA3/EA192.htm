Basic Path Basic Path
Details:
Steps:
1. DER data is sent to DER gateway<br/>
2. DER gateway sends data to tokenization tool<br/>
3. tokenization tool calculates data signature<br/>
4. tokenization tool stores data tokenization in blockchain<br/>
5. DER gateway sends data to DER SCADA<br/>
6. DER SCADA sends data to EMS<br/>
7. EMS sends data to tokenization tool<br/>
8. tokenization tool calculates tokenization data from raw data<br/>
9. blockchain is accesed by tokenization tool to retrieve stored tokenization data<br/>
10. tokenization tool sends the assessment of validity of data to EMS<br/>
11. rtu sends data to SCADA<br/>
12. SCADA sends data to EMS<br/>
13. EMS calculates required balancing commands<br/>
14. EMS sends command details to tokenization tool<br/>
15. tokenization tool tokenizes the data and stores in blockchain<br/>
16. EMS sends the command to DER SCADA<br/>
17. DER SCADA sends command to DER gateway<br/>
18. DER gateway sends command to tokenization tool<br/>
19. tokenization tool get tokenization onfo from blockchain and assess the validity<br/>
20. tokenization tool sends the command validity assessment to DER gateway<br/>
21. der gateway send command to DER<br/>
